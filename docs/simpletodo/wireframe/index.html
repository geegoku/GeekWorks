    3. ワイヤーフレームの作成

    ワイヤーフレームとは

    ワイヤーフレームとは、アプリの骨格や構造を表現したものです。(主に単色で、配色を指示しない表現物)

    ワイヤーフレームを作成することで、早い段階で機能の完成イメージ(=ゴール)を明確化できます。 例えるなら、山登り時のゴールを決めるのが、このフェーズです。

    ゴールが決まっていないと、変な道に脱線したり、遭難(失敗)する可能性が高まります。

    ToDoアプリの機能

    ワイヤーフレームの作成前に、まずは必要な機能・画面を抽出します。

    ToDoアプリの動作については、標準的なUIが w3schools.com(The To Do List) にあったので、これを参考サイトとします。これとマインドマップを元に、サイトマップを作成します。

    画面は一覧表示のみで、追加・更新・削除はリダイレクト処理をするので「画面無し」とします。 一覧化すると、以下のようになります。

    機能 Path リダイレクト 画面
    一覧表示 /index 無し 有り
    追加 /create 有り(一覧表示へ) 無し
    更新 /update 有り(一覧表示へ) 無し
    削除 /delete 有り(一覧表示へ) 無し
